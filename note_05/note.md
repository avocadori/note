## 第1章：なぜ「リサーチ自動化」が、初心者の最初の武器に最適なのか

### 理由①：誰もが困っている

リサーチ作業は、**ほぼすべての職種・業界の人が** 何らかの形で抱えています。

- マーケター → 競合チェック、市場動向、SNSモニタリング
- 営業 → 訪問前の企業情報リサーチ
- 経営者・企画 → 新規事業の市場調査
- ライター・編集 → ネタ収集、関連情報の確認
- 学生・研究者 → 論文や資料の要約

つまり、**「お客さんの数が、ほぼ無限」** という超恵まれた市場です。

### 理由②：成果が「数字」で見える

リサーチ自動化は、**時間削減量** で価値を示せます。

- 「毎週月曜の朝、競合チェックに2時間かかっていたのが10分になります」
- 「年間で換算すると、80時間の節約です」

これは見積もりが立てやすく、**説得力のある価格交渉** ができます。

### 理由③：技術的に「ちょうどいい難易度」

リサーチ自動化は、

- 完全ノーコード（Dify／Make／n8n）でも作れる
- 少しコードを書く（Cursor／Claude Code）と、さらに強くなる
- AIプロンプトの質で差別化できる（note_02 の知識がそのまま効く）

**「未経験から始めて、1ヶ月で形にできる」** ベストな複雑度です。

### 理由④：使い回しが効く

1つ作れば、入力データと出力先を変えるだけで **10種類のリサーチツールに化けます**。

- 競合A社の調査 → そのままB社にも使える
- ECサイトの価格調査 → 不動産物件の調査にも応用できる
- SNS分析 → ブログ分析にも転用できる

**1つの「型」が、複数の収益源になる**。これがリサーチ自動化の真骨頂です。

## 第2章：リサーチツールの基本構造｜4ステップに分解すれば、ぜんぶ作れる

どんなリサーチツールも、構造は同じです。たった4ステップ。

### ステップ①：収集（Collect）

「何を集めるか」を決める段階です。

- 競合サイトのページ
- ニュース記事のRSS（サイトが配信する更新情報の仕組み）
- SNSの投稿（X、Threadsなど）
- 自社の問い合わせメール
- Googleスプレッドシートのデータ

### ステップ②：整理（Clean）

集めたデータを、AIが扱いやすい形に整える段階です。

- HTMLタグ（Webページの裏側の記号）を取り除く
- 重複を消す
- 必要な部分だけを抜き出す

ここはノーコードツール（Dify／n8n）が自動でやってくれます。

### ステップ③：要約・分析（Summarize）

AIの本領発揮の段階です。

- 重要な情報を抜き出す
- 感情分析する
- カテゴリ分けする
- 前回からの変化を見つける

### ステップ④：出力（Output）

結果を、人が「使える形」に届ける段階です。

- Slackに通知
- メールで送信
- Notionに自動で書き出す
- スプレッドシートにレポートとして追加
- LINE Botで配信

### 重要なのは「3番のAI部分」が95%の価値を決める

集める・整える・出力するは、**ノーコードツールやAIコーディングで簡単に作れる** 工程です。差がつくのは、3番の **AIへの指示（プロンプト）** だけ。

## 第3章：3つの作り方｜あなたのレベルに合った道を選ぼう

リサーチツールの作り方は、大きく3パターンあります。

### 作り方①【軽量版】：Cursor or Claude にURLを渡すだけ

**難易度**：★ ★ ★ ★ ★（5段階で最易）
**所要時間**：5分
**できること**：手動で「特定のURLの情報をまとめて」と頼む

**手順**：

1. Cursor または Claude のチャット画面を開く
2. 以下のように頼む

```
以下のURLの内容を読んで、3行で要約してください。
そのあと、ビジネス的に注目すべきポイントを5つ挙げてください。

URL: {ここに調べたいURL}
```

これだけです。**「リサーチ自動化の入門」として、まずこれを毎日やる** だけでも、感覚がつかめます。

### 作り方②【ノーコード版】：Dify or n8n で定期実行

**難易度**：★ ★ ★（ややハードルあり）
**所要時間**：2〜4時間
**できること**：毎朝自動で実行→結果がSlackやメールに届く

**手順の概要**：

1. Dify または n8n のアカウントを作る（無料プランあり）
2. 「URLからデータを取る → AIに要約させる → 結果を送る」のフローを画面上で組み立てる
3. スケジュール設定（毎朝9時など）で自動実行
4. 出力先をSlack／メール／Notionに繋ぐ

このパターンは、**プログラミング不要で「毎朝届くニュースレターのようなもの」** が作れる、という大きな強みがあります。

### 作り方③【高機能版】：Cursor／Claude Code で本格構築

**難易度**：★ ★（少し本格的）
**所要時間**：半日〜1日
**できること**：複雑なロジック、Web画面、顧客への配布まで

**手順の概要**：

1. Cursor または Claude Code を起動
2. 「こんなリサーチツールを作って」と依頼（プロンプトは第4章のレシピを使う）
3. 動作確認しながら改良
4. Vercel／Netlify などの「公開用サービス」にアップ（これもAIに頼める）

このパターンは、**サブスク型サービス（毎月の利用料を払って使うサービス。NetflixやSpotifyと同じ仕組み）として売れる完成度** のものが作れます。

<!-- 📌 画像挿入位置 4：three_builds.png（3つの作り方の難易度・成果対比） -->

### どれから始めるべき？

**完全未経験** → 作り方①（軽量版）を1週間。リサーチ感覚を身につける。
**少し慣れた** → 作り方②（ノーコード版）に挑戦。受託案件で売れるレベル。
**3ヶ月以上やる予定** → 作り方③（高機能版）。サブスク型サービスとして月額収入を狙う。

## 第4章：そのまま使える！5つのリサーチレシピ

ここからは、コピペで使えるレシピ集です。プロンプト部分は、AIにそのまま貼れば動きます。

### レシピ①：競合サイトの自動定期チェッカー

**対象**：EC運営者、メディア運営者、マーケター
**目的**：競合の新商品・価格変動・キャンペーンを毎朝ダイジェストで受け取る

**作り方②（n8n）の例**：

1. URL（最大5社分）をリスト化
2. n8nで「毎朝7時に各URLからHTMLを取得」
3. 取得した内容を以下のプロンプトでAIに渡す

```
# 役割
あなたは競合調査のプロです。

# 指示
以下のWebページの内容を読んで、前回のスナップショットと比較し、
「新しい変化（新商品／値段／キャンペーン／お知らせ）」だけを抽出してください。

# 入力
【今回のページ内容】
{今回取得したHTMLや本文}

【前回のページ内容】
{前回保存したもの}

# 出力形式
- 新しく追加された項目（箇条書き）
- 価格や数値が変わった項目（前→後）
- 重要度（高／中／低）
- 注目すべきポイント（3行で）
```

4. 結果をSlack or メールで毎朝配信

**収益化**：1社あたり月3〜5万円のサブスク型サービス／受託で5〜15万円の案件として成立

---

### レシピ②：業界ニュースの毎朝ダイジェスト

**対象**：経営者、企画担当、業界研究者
**目的**：自分の業界に関連するニュースを、AIが選別して毎朝届ける

**作り方②（Dify）の例**：

1. 業界キーワードを設定（例：「生成AI」「SaaS」「B2B（企業間取引）」など）
2. ニュースAPI（NewsAPI、Yahoo!ニュースなど）から記事を毎朝取得
3. プロンプト：

```
# 役割
あなたは業界アナリストです。

# 指示
以下のニュース記事の中から、{業界名}に関連する重要なものを5本選び、
それぞれ要約してください。「自分が経営者なら知っておきたい」基準で選別。

# 入力
【記事リスト（タイトル＋本文の冒頭）】
{ここに記事リスト}

# 出力形式
重要度順に5本。
- タイトル
- 要約（3行）
- 自社への含意（1行）
- 元URL
```

4. 結果をメール or Notionに自動投稿

**収益化**：個人向け月額500〜1,500円のニュースレター／企業向け月3万円〜

### レシピ③：SNS口コミの自動収集ツール

**対象**：商品企画、マーケティング、カスタマーサポート
**目的**：自社や競合に関するSNSの声を、定期的に集めて分析

**作り方③（Cursor）の例**：

Cursorに以下のように頼みます。

```
SNS（X）で特定キーワードの投稿を集めて、AIで分析するツールを作ってください。

【入力】
- 監視キーワード（複数登録可能）
- 取得期間（直近24時間／7日 など）

【処理】
1. キーワードを含む投稿を取得（X API使用）
2. 各投稿の感情（ポジティブ／ニュートラル／ネガティブ）を判定
3. 共通テーマを3つ抽出
4. 改善提案を3つ生成

【出力】
- 一覧画面（ダッシュボード）形式のHTMLレポート
- 感情の割合（円グラフ）
- 主要なテーマ
- 注目すべき投稿（5件）

【技術】
- 画面側はWebアプリ（HTML/CSS/JS）
- 裏側の処理は Vercel のサーバーレス関数（必要なときだけ動く小さなプログラム）
```

このプロンプトをそのまま投げれば、AIが必要な構成を返してくれます。あとは指示しながら改良するだけ。

**収益化**：1社あたり月5〜10万円のサブスク型サービス／単発リサーチ案件で10〜30万円

### レシピ④：顧客の声（VOC）の一括分析

**対象**：商品企画、カスタマーサポート、経営層
**目的**：問い合わせメールやアンケートを一括でAIに分析させる

**作り方①or②**：

```
# 役割
あなたはVOC（Voice of Customer）分析の専門家です。

# 指示
以下の顧客の声リストを分析してください。
1. 感情（ポジティブ／中立／ネガティブ）の割合
2. よく挙がるテーマ（カテゴリ3〜5個）
3. それぞれのテーマで「最も切実な声」を1〜2件抜粋
4. 「すぐに対応すべき改善提案」を3つ
5. 「中長期で取り組むべきテーマ」を2つ

# 入力
【顧客の声リスト】
{ここに問い合わせやアンケート結果を貼る}

# 出力形式
レポート形式。グラフ用の数値表もつける（マークダウンの表で）。
```

**収益化**：1案件10〜30万円のスポット受託／継続契約なら月5〜10万円

## 第5章：リサーチプロンプトの「5つのコツ」

リサーチタスク特有のプロンプトのコツです。これを知っているだけで、結果の質が3倍変わります。

### コツ①：「目的」を最初に伝える

```
❌ このページを要約して。
⭕ 自社の新商品企画の参考にしたい。このページから「ユーザーが本当に求めている機能」を抜き出して。
```

**目的が明確だと、AIは関係ない情報を捨てて要点に絞ります。**

### コツ②：「比較の軸」を指定する

```
3社のサービスを、以下の軸で比較してください。
- 価格
- 対象顧客
- 強み
- 弱み
- 公式の言い回し
```

軸を決めると、出力が **そのまま意思決定に使える形** になります。

### コツ③：「数字で出して」と頼む

定性的な情報を、数値化させると意思決定に使いやすい。

```
このサービスを、以下の5項目で10点満点で採点してください。
- 価格の安さ
- 機能の充実度
- 使いやすさ
- サポート
- 将来性

採点理由も1行ずつ添えてください。
```

### コツ④：「変化」を見つけさせる

リサーチは「今」より「前回との差分」が価値を持ちます。

```
前回（{日付}）と比べて、以下の変化を抜き出してください。
- 新しく追加されたもの
- 削除されたもの
- 数値が変わったもの
- 表現が変わったもの
```

### コツ⑤：「次のアクション」まで提案させる

ただ整理しただけのレポートは、価値が低い。

```
以上の分析を踏まえて、私たちが取るべき次のアクションを3つ提案してください。
それぞれ「短期（1ヶ月以内）／中期（3ヶ月）／長期（半年〜1年）」で。
```

これだけで、レポートは **「読まれる資料」から「動かす資料」** に変わります。

## 第6章：リサーチツールの収益化｜どう売るのが現実的か

ここまで作れるようになったら、お金にする番です。3つの王道パターンを紹介します。

### パターンA：単発リサーチ案件（5〜30万円／件）

「うちの業界で◯◯について調べてくれ」という依頼を受ける形式。

- **どこで取る**：ココナラ、ランサーズ、X、知人紹介
- **強み**：1件で大きな金額が動く、納品が早い
- **弱み**：時間を切り売り、リピートしにくい
- **おすすめ度**：★★★★（最初の収益化はここが最短）

### パターンB：継続レポート契約（月3〜10万円／社）

「毎週／毎月、これを定期的に届けてくれ」という形式。

- **どこで取る**：単発案件のお客さんを、継続契約に切り替える
- **強み**：安定収入、関係が深まる
- **弱み**：作り込みが必要、初動が遅い
- **おすすめ度**：★★★★★（10万円安定の最短ルート）

### パターンC：サブスク型サービス化（月額500〜10,000円）

「誰でも使えるツール」として公開する形式。

- **どこで売る**：LP（ランディングページ）＋SNS＋note記事
- **強み**：寝てる間も売れる、スケールが大きい
- **弱み**：マーケが大変、競合も多い
- **おすすめ度**：★★★（半年以降に挑戦）

### 結論：A → B → C の順に育てる

最初は **A（単発受託）** で実績と現金を作る。同じお客さんを **B（継続）** に変える。Bが何社か貯まったら、共通部分を **C（サブスク型サービス）** に抽出する。

これがリサーチ自動化で稼ぐ人の鉄板ルートです。

## 第7章：リサーチ自動化でよくある失敗

### 失敗①：AIの出力をそのまま信用してしまう

リサーチでは、AIが「もっともらしい嘘」を混ぜることが時々あります（特にURLや数字）。

**回避策**：重要な数字や引用は **必ず元ソースを確認する**。出力に「出典URL」を必須項目として含めるとよい。

### 失敗②：取得対象のサイトの利用規約を読まない

スクレイピング（ページの自動取得）は、サイトによっては禁止されています。

**回避策**：取得前にサイトの利用規約と、robots.txt（サイト側が「自動取得していい範囲」を示すルールファイル）を確認。公式APIがあるならそちらを使う。

### 失敗③：データを保存して情報漏洩リスクを背負う

顧客の声や社内データを扱う場合、保管・通信の安全性が問われます。

**回避策**：

- 顧客データはローカル or 安全なクラウドのみ
- AIに渡すデータは、機密項目をマスキング
- 業務委託契約書に「データ取り扱い」の条項を入れる

### 失敗④：「何でも分析できます」と売ろうとする

汎用ツールは「誰にも刺さらない」状態になりやすい。
