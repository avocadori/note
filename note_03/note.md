## 第1章：2026年、AIコーディングは「書く」から「頼む」へ

### プログラミングの常識が、この2年で完全にひっくり返った

2023年までのプログラミングは、こんな世界でした。

- 文法を覚える → 構文エラーで詰まる → 検索する → スタックオーバーフローで答えを探す → コピペして直す
- 1つの機能を作るのに数日かかる
- エラー対応が大半の時間を占める

しかし2026年の今、状況は完全に変わっています。

- **日本語で「ログイン機能を追加して」と頼むだけ** で、AIが10秒で書く

- **「もっとオシャレな見た目にして」「スマホ対応にして」** など、抽象的な依頼にも応えてくれる

つまり、**「書く技術」より「頼む技術」が重要な時代** に入りました。

## 第3章：Cursorではじめる「自然言語コーディング」入門

ここから実践編です。Cursor を例に、実際の開発の流れを見ていきます。

### ステップ1：Cursorをインストールする

1. [cursor.com](https://cursor.com) にアクセス
2. 自分のOS（Mac／Windows）に合うものをダウンロード
3. インストールして起動

これだけです。プログラミング言語のインストールなどは、後から必要なときにAIが教えてくれます。

### ステップ2：新しいフォルダを開く

Cursorを起動して、適当な場所に「my-first-app」のような新しいフォルダを作って開きます。

### ステップ3：AIに「作ってほしいもの」を伝える

`Cmd + L`（Macの場合）または `Ctrl + L`（Windowsの場合）を押すと、AIに話しかけるチャット画面が開きます。

そこに、こう打ち込んでみます。

```
シンプルなTODOアプリを作ってください。

- HTMLとCSSとJavaScriptだけで完結すること
- タスクを追加・完了・削除できる
- ブラウザでindex.htmlを開くだけで動くこと
- 見た目は、白基調でやさしいパステル調
```

15〜30秒で、AIが必要なファイルを全部作ってくれます。

### ステップ4：実際に動かす

作られた `index.html` をダブルクリックすると、ブラウザで動きます。**プログラミングを書かずに、最初のアプリが完成しました。**

### ステップ5：改造する

もう一度AIに話しかけます。

```
完了したタスクに取り消し線をつけてください。
あと、追加ボタンを押したときに軽くアニメーションさせてください。
```

これだけで、AIが既存のコードを正しく書き換えてくれます。

### Cursorで使う「3つの便利機能」

| ショートカット | できること |
|----------------|-----------|
| `Cmd + L` | チャットでAIに依頼（コード全体を読んだ上で答えてくれる） |
| `Cmd + K` | コードを選択して「ここをこう変えて」と指示 |
| `Cmd + I`（Composer） | プロジェクト全体にまたがる大きな変更を依頼 |

特に **Composer（コンポーザー）** は、新しい機能を追加するときの主役です。「ユーザー登録機能を追加してください」と書けば、複数ファイルにまたがる変更を一気にやってくれます。

## 第4章：Claude Codeで「ターミナルから対話する」開発

Cursorに慣れてきたら、次の選択肢が **Claude Code** です。

### Claude Codeとは何か

ターミナル（黒い画面）からClaudeと対話しながら開発を進める、CLI（コマンドラインインターフェース）型のAIアシスタントです。

「ターミナルって難しそう……」と思うかもしれませんが、**やることはCursorとほぼ同じ** です。違いは「画面が違うだけ」。慣れると、こちらの方が爆速になります。

### Claude Codeでできること

- 自然言語でファイルの編集を依頼
- 「テストを書いて、実行して」と頼むと自動で進めてくれる
- 「Gitにコミットして」と頼むと、適切なコミットメッセージまで作る（Git＝コードのバージョン管理ツール、コミット＝変更を1つの区切りとして保存すること）
- プロジェクト全体を理解した上で整理整頓（リファクタリング）できる

### 使い方の例（イメージ）

ターミナルを開いて、プロジェクトのフォルダに移動。Claude Codeを起動して、こう話しかけます。

```
ユーザー登録ページを追加してください。
データはひとまずブラウザに保存して、後でデータベース連携できる構造にしておいてください。
```

すると、Claudeが必要なファイルを作り、修正し、確認のために動かしてくれます。エラーが出たら自動で原因を探って直してくれます。

### Claude CodeとCursorの使い分け

| シーン | おすすめ |
|--------|---------|
| 画面を見ながらちょこちょこ修正したい | Cursor |
| 大きな機能をまとめて作りたい | Claude Code |
| エラー対応を自動でやってほしい | Claude Code |
| ボタンで操作する画面（GUI）が好き | Cursor |
| キーボードだけで完結したい | Claude Code |

## 第5章：自然言語で上手に頼む「7つのコツ」

前回のnoteで学んだプロンプトエンジニアリングの応用編です。AIコーディングで特に効くコツを7つにまとめました。

### コツ①：目的を最初に伝える

```
❌ ボタンを赤くして。
⭕ ユーザーに「危険な操作」だと気づかせたいので、削除ボタンを赤くしてください。
```

「なぜそうするのか」を伝えると、AIは前後の文脈も考えて整えてくれます。

### コツ②：制約をはっきり書く

```
❌ ログイン機能を作って。
⭕ ログイン機能を作ってください。
   - ライブラリは増やさず、既存のものだけで実装
   - メールアドレスとパスワードのみ
   - 試作なのでデータはブラウザに保存でOK
```

「やってほしくないこと」を書くと暴走を防げます。

### コツ③：1度に頼むのは1つだけ

5つの機能をまとめて頼むと、AIが混乱します。**「1機能ずつ、確認しながら進める」** が鉄則です。

### コツ④：エラーは「全文」を渡す

エラーが出たら、エラーメッセージを **省略せずそのまま** AIに見せる。一部だけ渡すと診断がブレます。

### コツ⑤：迷ったら「2つ案を出して」

```
このログイン機能、2つの実装案を出して、それぞれのメリット・デメリットを教えてください。
```

AIが「最初に思いついた1案」より、比較した上で選ぶ方が圧倒的に良い設計になります。

### コツ⑥：「説明してから書いて」と頼む

```
変更する前に、何をどう変えるかを3行で説明してから、実際のコードを書いてください。
```

これでAIの暴走を防げます。先に方針を確認できるので、納得した上でコードに反映できます。

### コツ⑦：「もう一度自分でチェックして」と頼む

```
書き終わったら、もう一度自分で読み返して、バグや問題点がないか確認してください。
```

これだけで品質が体感1.5倍。AIは「ダブルチェック」を頼むと素直にやってくれます。

## 第6章：コピペで試せる！実例レシピ集（5個）

ここからは、そのまま使えるレシピ集です。Cursor または Claude Code のチャット欄に貼るだけで、動くアプリが手に入ります。

### 🔰 レシピを読む前に｜出てくる用語を3分で確認

以下のレシピには技術用語が出てきますが、**意味はAIが理解してくれます**。あなたが全部理解する必要はありません。一応、ざっくりした意味だけ書いておきます。

| 用語 | やさしい意味 |
|------|-------------|
| HTML / CSS / JavaScript | Webサイトを作るための3つの言語。AIが書いてくれるので暗記不要 |
| API（エーピーアイ） | AIと自分のアプリをつなぐ仕組み（例：OpenAI API＝ChatGPTを呼び出す） |
| ブラウザの保存機能（localStorage） | サイトの情報を、ブラウザを閉じても残す仕組み |
| Vercel（ヴァーセル） | アプリを無料でネットに公開できるサービス |
| サーバーレス関数 | 裏側で動く小さなプログラム。ボタンを押したときに自動で実行される |
| UI（ユーアイ） | 画面の見た目・操作する部分 |

これだけ知っておけば、レシピのプロンプトをそのままAIに貼っても、ちゃんと意図通りのアプリが返ってきます。

### レシピ①：シンプルなTODOアプリ

```
HTMLとCSSとJavaScriptだけで完結する、シンプルなTODOアプリを作ってください。

【機能】
- タスクの追加
- 完了マーク（取り消し線）
- 削除
- ブラウザを閉じても残るようにブラウザの保存機能（localStorage）を使う

【見た目】
- 白基調にパステルカラーのアクセント
- スマホでも見やすい

【ファイル】
- index.html
- style.css
- app.js

完成したら、index.htmlを開いて動作確認できる状態にしてください。
```

### レシピ②：見積もり自動計算ツール

```
小さな個人事業主向けの「見積もり自動計算ツール」を作ってください。

【入力項目】
- 商品名（追加・削除可能）
- 単価
- 数量
- 値引率（％）

【計算結果】
- 小計
- 値引額
- 消費税（10%）
- 合計

【機能】
- PDFで保存できる
- 印刷できる

【技術】
- HTML/CSS/JavaScript のみ
- ライブラリ追加は最小限に
```

### レシピ③：AIチャットボット（OpenAI API使用）

```
OpenAI APIを使った、シンプルなチャットボットを作ってください。

【動作】
- ブラウザで動く
- 入力欄にメッセージを書いて送信ボタンを押すと、AIから返事が返ってくる
- 過去の会話履歴も画面に残る

【設定】
- APIキーは、画面上で入力する欄を設ける（コードには直接書かない）
- 入力したキーはブラウザの保存機能（localStorage）に保存

【見た目】
- LINEのようなチャットUI
- スマホ対応
```

### レシピ④：URLからの自動要約ツール

```
URLを入力すると、そのページの内容を自動で要約してくれるツールを作ってください。

【動作】
- URLを貼り付けて「要約する」ボタンを押す
- 5行以内に要約された結果が表示される
- 要約結果をコピーするボタンつき

【技術】
- 画面側は HTML/CSS/JavaScript で作る
- 要約はOpenAI API（ChatGPTを呼び出す仕組み）を使う
- ページの本文を取りに行く処理は、ブラウザから直接できない制約があるため、
  Vercelのサーバーレス関数（裏側で動く小さなプログラム）を使った構成にしてください
```

### レシピ⑤：日記アプリ（音声入力対応）

```
スマホで使える、音声入力できる日記アプリを作ってください。

【機能】
- マイクボタンを押して話すと、文字起こしされる（ブラウザの音声認識機能を使用）
- 文字起こし結果は手動でも修正できる
- 保存すると日付つきで履歴に残る
- 過去の日記を一覧で見られる

【見た目】
- やさしい雰囲気（パステル、丸み）
- スマホでの使いやすさを最優先

【技術】
- HTML/CSS/JavaScript のみで完結
- 保存はブラウザの保存機能（localStorage）
```

## 第7章：初心者が陥りがちな失敗パターン

### 失敗①：いきなり大きすぎる依頼をする

「ECサイトを作って」「SNSを作って」など、巨大な要求から始めると、AIも迷子になります。

**回避策**：機能を **「TODOアプリ」レベルに小さく** 分解する。最初は「タスク追加機能だけ」のように、超ミニマムから。

### 失敗②：エラーが出たらAIに何も伝えずに諦める

エラーは AIにとって最高のヒントです。コピペして渡せばすぐ直してくれます。

**回避策**：エラーメッセージは **省略せず全文をそのまま** AIに見せる。

### 失敗③：AIの出すコードを「全部信用」してしまう

AIは時々、存在しないライブラリ名を書いたり、古い書き方をしたりします。

**回避策**：1機能できるたびに、必ず動作確認する。動かなかったら「動かない」と伝える。

### 失敗④：自分のPCに何も入れずに永遠に消費する

「もう少し勉強してから」と言って、CursorもReplitも触らないまま1ヶ月。これが一番もったいない。

**回避策**：このnoteを読み終わった今夜、**最低1つはツールをインストールする**。動くものを作る感覚は、触ってみないと永遠に得られません。
